# 大乐透8红三球覆盖独立验证 Implementation Plan（2026080）

> **执行约束：** 本计划在当前任务内顺序执行；不派发子代理，不执行未经用户许可的Git操作。

**Goal:** 在不读取封存窗口表现选参的前提下，用2025001—2026041开发数据选出唯一时间分块集成冠军；冻结后仅开启一次2026042—2026079独立验证，并依据14/38晋级门槛决定是否替换当前均衡V2。

**Architecture:** 主计算分为互相隔离的`develop`和`validate`两阶段。`develop`物理截断至2026041，生成基础策略池、三个时间块指标、Top 3/5/7/9集成和唯一冠军；冠军参数先写入冻结记录并计算SHA-256。只有冻结记录存在且带有`VALIDATION_NOT_OPENED`时，才允许`validate`载入2026042—2026079。最终再用另一种语言独立重算逐期结果。

**Tech Stack:** Node.js内置模块、Python 3标准库、Markdown、`data/da_2025_data.json`、`data/da_2026_data.json`。

## Global Constraints

- 2025数据预期150期、2025001—2025150；2026数据预期79期、2026001—2026079。
- 开发可见上限固定为2026041；开发滚动目标为2025039—2026041，共153期。
- D1/D2/D3分别为2025039—2025089、2025090—2025140、2025141—2026041，各51期。
- 封存验证固定为2026042—2026079，共38期；开发阶段不得计算其候选表现。
- 8红最多2个上期前区重号；3胆码最多1个上期前区重号。
- 蓝球沿用均衡V2固定模型，不参与前区策略筛选。
- 晋级门槛固定为8红覆盖至少3个达到14/38；未达标不得用封存期再次调参。
- 2026080不得存在于数据中，也不得进入训练、开发、验证或调参。
- 不修改开奖JSON，不执行提交、推送、合并等Git操作。

### Task 1: 数据边界与开发阶段执行器

**Files:**
- Read: `data/da_2025_data.json`
- Read: `data/da_2026_data.json`
- Create: `scripts/dlt_cover3_time_block_ensemble.mjs`

**Interfaces:**
- `node scripts/dlt_cover3_time_block_ensemble.mjs develop`
- Consumes: 两份原始开奖JSON，但开发模式只保留期号不晚于2026041的记录。
- Produces: 开发候选指标、四个集成指标、冠军参数和冻结所需JSON；不得输出任何封存验证命中指标。

- [ ] **Step 1: 先写数据质量断言**

  检查期号顺序、前区5个/后区2个号码、范围、升序、唯一性和前区和值；断言总计229期、开发191期、开发目标153期、三个时间块各51期、封存38期、目标期2026080不存在。

- [ ] **Step 2: 实现纯前向评分函数**

  每次预测只接收目标期之前的`history`。实现多窗口频率、四种遗漏形态、分区/奇偶/和值状态、上期共现和非正向重号项；最新一期频率计数按0.5，遗漏0按有效遗漏1处理。同分按号码升序。

- [ ] **Step 3: 预生成基础策略池**

  候选ID和参数按固定顺序生成，覆盖：短窗6种、中窗2种、长窗3种、频率权重3种；遗漏形态4种；最近5/10/长期分区与奇偶状态；L/M/H/X状态频率；共现聚合和重号权重`-0.10/-0.05/0`。候选定义完成后固定，不依据封存38期增删。

- [ ] **Step 4: 统一两阶段选码**

  所有候选从完整01—35排名中先选3胆（重号≤1），再从剩余排名选5拖（8红总重号≤2）；输出完整排名供后续集成投票。

- [ ] **Step 5: 增加开发模式泄漏防护**

  `develop`模式若任何评价目标大于2026041立即失败；输出中必须包含`VALIDATION_NOT_OPENED`，且不得出现2026042—2026079逐期预测或命中字段。

### Task 2: 开发滚动、稳定性排序与冠军冻结

**Files:**
- Read: `scripts/dlt_cover3_time_block_ensemble.mjs`
- Create: `docs/lottery/大乐透8红三球覆盖开发冻结记录_2026080.md`
- Create: `docs/lottery/大乐透8红三球覆盖开发冻结记录_2026080.json`

**Interfaces:**
- Consumes: Task 1开发模式输出。
- Produces: 唯一冻结冠军、Top 3/5/7/9开发指标、完整参数摘要和校验哈希。

- [ ] **Step 1: 对基础策略执行153期严格滚动**

  对每个开发目标只使用此前历史，分别累计D1/D2/D3和整体的覆盖≥1/≥2/≥3、总覆盖、胆码≥1/≥2及最大重号数。违反重号上限的候选直接淘汰。

- [ ] **Step 2: 按锁定词典序排序**

  依次比较最低块覆盖≥3率、块间方差（越小越优）、整体覆盖≥3、总覆盖、覆盖≥2、覆盖≥1、胆码≥2、胆码≥1、复杂度和候选ID，得到固定基础策略排名。

- [ ] **Step 3: 构造Top 3/5/7/9集成**

  对每个目标期重新计算入选基础策略排名，使用`1/(1+rank)`平均投票并执行相同重号约束；四个集成仍按同一词典序选择唯一开发冠军。

- [ ] **Step 4: 写入冻结记录**

  记录数据范围、候选总数、基础策略排名、四个集成开发指标、冠军完整参数、三个时间块指标和`VALIDATION_NOT_OPENED`。JSON冻结文件作为机器复核输入，Markdown冻结文件作为人工证据。

- [ ] **Step 5: 计算并回填SHA-256**

  对规范化冠军参数JSON计算SHA-256；两个冻结文件中的冠军ID、参数、集成规模和摘要必须一致。完成后重新运行开发模式，确认同一输入得到同一摘要。

### Task 3: 单次开启封存38期独立验证

**Files:**

**Interfaces:**
- `node scripts/dlt_cover3_time_block_ensemble.mjs validate docs/lottery/大乐透8红三球覆盖开发冻结记录_2026080.json`
- Consumes: 已冻结冠军和完整历史数据。
- Produces: 2026042—2026079共38行逐期结果、汇总指标和晋级判定；验证阶段禁止改变冠军参数。

- [ ] **Step 1: 验证冻结前置条件**

  校验冻结JSON仍带`VALIDATION_NOT_OPENED`、冠军摘要匹配、参数结构完整、开发边界不变；任一失败都停止验证。

- [ ] **Step 2: 执行38期严格滚动**

  每期仅使用该期之前的全部历史生成3胆、5拖、8红和2蓝；记录胆码≥1/≥2、8红覆盖≥1/≥2/≥3、红球总覆盖、2蓝命中、扩展联合、8红重号数和胆码重号数。

- [ ] **Step 3: 固化验证结果且禁止重开调参**

  将状态从`VALIDATION_NOT_OPENED`改为`VALIDATION_OPENED_ONCE`，写入38行和汇总；保留原冠军参数与摘要。后续执行只允许复核相同结果，不允许重新搜索候选。

- [ ] **Step 4: 做一次性晋级判定**

  若覆盖≥3不少于14/38且两类重号约束全通过，标记`PASS_PROMOTE_V3`；否则标记`FAIL_KEEP_V2`。无论结果如何，都如实保留独立验证证据。

- [ ] **Step 1: 成功路径**

  明确38期封存验证不是未来概率保证；彩票为独立随机事件，模型只用于统计实验与小额娱乐。
