# 大乐透重号均衡优化 Implementation Plan

> **For agentic workers:** 本计划在当前任务内顺序执行；工作区规则不允许自行派发子代理，也不允许未经用户许可执行Git操作。

**Goal:** 将已确认的均衡V2重号控制规则应用于大乐透严格滚动模型，重新生成2026080预测并同步综合复用文档和期号专属报告。

**Architecture:** 保留四状态基础窗口、分区算法和后区模型，只替换前区的最新一期计数、遗漏0期处理、显式重号权重及两阶段选码约束。主计算使用Node.js从原始JSON逐期物理截断；文档完成后使用独立Python实现重新计算并逐行核对。

**Tech Stack:** Node.js内置模块、Python 3标准库、Markdown、`data/da_2026_data.json`。

## Global Constraints

- 数据只允许来自`data/da_2026_data.json`，当前预期为79期、2026001—2026079。
- 目标期固定为数据最新期的下一期，即2026080；数据中不得存在2026080。
- 最新一期在前区短窗和长窗中按0.5次计数，窗口分母不变。
- 前区有效遗漏使用`max(实际遗漏, 1)`。
- V2显式重号权重为L=0、M=0、H=0、X=-0.1。
- 3胆码最多1个上期重号；8红合计最多2个上期重号。
- 后区模型、四状态边界和综合分同分升序规则保持不变。
- 最近38期必须逐期物理截断，不能使用当期及未来数据。
- 不搜索0.5系数、遗漏下限或重号权重，不为命中率反向调参。
- 不修改历史开奖JSON，不执行提交、推送、合并等Git操作。

### Task 2: 应用均衡V2并生成严格滚动结果

**Files:**
- Read: `data/da_2026_data.json`
- No persistent analysis code is created.

**Interfaces:**
- Consumes: Task 1的`draws`、四状态基础参数、V2结构规则。
- Produces: 38行V2逐期结果、汇总指标、2026080评分排名和最终号码。

- [ ] **Step 1: 实现最新一期半权频率**

  对前区每个窗口使用`(除最新期外出现次数 + 0.5 × 最新期出现标记) / 窗口期数`；后区继续使用原始频率。

- [ ] **Step 2: 实现遗漏和显式重号修订**

  前区遗漏项使用`max(gap, 1)`；L/M/H显式重号权重设为0，X保持-0.1。

- [ ] **Step 3: 实现两阶段选码**

  第一阶段从完整排名选择3胆码且胆码重号≤1；第二阶段从完整排名重新扫描、排除胆码，选择5拖码且8红重号总数≤2。记录胆码阶段暂缓重号和总上限正式跳过重号。

- [ ] **Step 4: 执行38期严格滚动回测**

  对2026042—2026079逐期只使用当期之前数据，累计胆码≥1/≥2、覆盖≥1/≥2/≥3、总覆盖、蓝球和扩展联合，并统计胆码与8红重号分布。

- [ ] **Step 5: 生成2026080预测**

  使用2026001—2026079全部可见数据，输出3胆码、5拖码、8红全集、2蓝、入选重号、胆码暂缓重号和总上限正式跳过重号。

- [ ] **Step 6: 验证V2结构约束**

  对38期和2026080分别断言`dan_repeat <= 1`与`all_repeat <= 2`；任何一期不满足都视为实现失败，不允许仅在文档中修正。
