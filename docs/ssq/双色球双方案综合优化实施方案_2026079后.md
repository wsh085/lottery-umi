## 一、目标、边界与整改背景

本方案统一管理两类目标：

1. 红球每期输出2个不同胆码，当期6个红球包含任意一个即命中。
2. 蓝球每期输出4个不同围选号，实际蓝球落入四码集合即命中。

本次整改针对上一版审阅发现的六项问题：

1. 红球融合版38期15/38，低于旧规则方案20/38，却被直接描述为优化版。
2. 蓝球38期13/38的95%区间仍包含25%随机基线，不能称为“明显高于”。
3. softmax输出总和为1，但没有独立校准集和可靠性指标，不能称为“概率已校准”。
4. Temperature不会改变Top4排序，不应放入以Top4命中率为目标的参数搜索。
5. 原无前视测试在调用前已经截掉未来数据，测试边界不够真实。
6. 文档遗漏了ML训练只回看最近72个目标点、数据指纹及参数冻结状态。

双色球开奖应视为独立随机事件。模型只能改善历史数据处理、比较和复盘方式，无法建立确定性开奖规律。

## 三、红球Champion/Challenger架构

### 3.1 Champion：严格规则自适应模型

Champion沿用原红球严格滚动方案：

1. 六个固定策略：近期热号、适中遗漏、冷热混合、重号延续、分区回补、结构均衡。
2. 每个目标时点只使用目标期以前数据。
3. 分别统计六策略此前最近12期和24期的严格滚动命中。
4. 排名分固定为：

```text
RankScore = 0.7×近12期命中率 + 0.3×近24期命中率
```

5. 前三策略按3、2、1权重投票。
6. 票数相同时使用固定综合分、遗漏距离和号码升序裁决。

当前Champion预测为`01、24`。

### 3.2 Challenger：规则投票与逻辑回归融合

Challenger对每个红球01—33构造：

- 最近5/10/20/50/100期频率；
- 当前遗漏和适中遗漏得分；
- 上期重号及邻近程度；
- 近12期三区偏差和奇偶偏差。

逻辑回归只使用预测时点以前最近72个历史目标点训练。最终融合分为：

```text
FinalScore_red = 0.65×逻辑回归归一化分 + 0.35×规则投票归一化分
```

当前Challenger预测为`09、24`。

### 3.3 晋级规则

Challenger当前不自动晋级。晋级至少需要：

1. 冻结当前结构和权重后，在新增开奖构成的前向审计中持续不低于Champion。
2. 同期逐期配对比较中，Challenger独中期数不低于Champion独中期数。
3. McNemar配对检验或更长前向样本支持差异，而不是只看一个累计命中率。
4. 不通过改变命中定义、窗口或数据起点取得优势。

## 四、蓝球softmax归一化评分模型

### 4.1 模型组成

蓝球模型对01—16每个号码计算21项历史特征，包括：

- 最近3/6/12/24/48期频率；
- 遗漏、适中遗漏和时间衰减；
- 上期重号和邻号；
- 近6/12期均值邻域和值域邻域；
- 近8期距离和邻号密度；
- 短长窗口冷热切换；
- 再激活分、分区偏差和奇偶偏差。

机器学习部分使用最近72个历史目标点训练共享候选评分器。最终综合分：

```text
Score_blue(n) = BayesWeight×贝叶斯归一化分
              + MLWeight×机器学习归一化分
              + 0.08×规则投票归一化分
              + ZoneBonus
```

当前默认参数：

```text
BayesWindow = 30
PriorStrength = 4.0
MLWeight = 0.15
BayesWeight = 0.77
LegacyWeight = 0.08
Temperature = 0.9（固定，不参与Top4命中率搜索）
```

综合分经过softmax归一化后总和为1，再选得分最高的4个号码。当前输出为`01、02、04、08`。

### 4.2 为什么不再称为“概率已校准”

softmax只把分数转换为总和为1的分布，不能自动保证：

- 预测10%的事件长期约有10%发生；
- 输出概率具有良好的Log Loss或Brier Score；
- Temperature经过独立校准集拟合；
- 概率分桶与真实频率一致。

因此代码模型名改为`blue_multiclass_softmax_normalized_auto_tuned`，并输出状态`softmax_normalized_not_calibrated`。

### 4.3 参数搜索整改

原参数网格为3×3×3×4，共108组，其中Temperature不会改变Top4排序。整改后参数网格缩减为：

```text
BayesWindow ∈ {24, 30, 36}
PriorStrength ∈ {4.0, 6.0, 8.0}
MLWeight ∈ {0.15, 0.22, 0.30}
```

共27组，Temperature固定为0.9。

每个目标期只在该期以前的12/24/36期内部窗口评价参数。候选必须在三个窗口的Top4覆盖率上都至少高于默认参数9个百分点才切换。这里的9个百分点只是工程阈值，不称为统计显著性。

### 4.4 参数冻结状态

- 当前参数冻结节点：2026079期后。
- 2026079及以前结果已经参与模型开发和审阅，属于开发证据。
- 2026080及以后新开奖才可作为冻结后的真正前向审计。
- 后续如改特征、权重或搜索网格，应重新标记模型版本，不能继续累加到旧版本审计成绩。
