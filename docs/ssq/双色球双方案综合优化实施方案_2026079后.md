## 一、目标与边界

本次工作不是直接“换一组更好看的号码”，而是把两套现有方案放进统一、可复现、可审计的优化框架中，完成以下落地：

1. 系统诊断两套方案的逻辑漏洞、数据局限与模型缺陷。
2. 基于公开可追溯历史开奖数据，补齐统一的数据校验、特征工程、严格前视回测和概率验证体系。
3. 将红球两胆方案与蓝球四码方案分别升级，并补充一套融合机制。
4. 交付可执行脚本与单元测试，而不是只停留在方法论层面。

需要强调：双色球开奖应视为独立随机事件，任何优化都只能改善“历史样本上的解释与筛选框架”，不能把历史命中率直接解释为未来真实胜率。

## 二、现有两套方案梳理

| 方案         | 目标                          | 核心方法                                                | 历史口径               | 文档记录结果     |
| ------------ | ----------------------------- | ------------------------------------------------------- | ---------------------- | ---------------- |
| 红球两胆方案 | 预测2个红球胆码 + 2个蓝球胆码 | 6个固定策略，按最近12/24期表现自适应排序，再做3/2/1投票 | 最近38期严格滚动       | 红球20/38，52.6% |
| 蓝球四码方案 | 仅预测4个蓝球围号             | 6个固定蓝球策略，内部12/24期排序后投票，再做三区覆盖    | 最近20/30/38期严格滚动 | 38期8/38，21.1%  |

两套方案共同点：

- 都强调严格前视，不允许读未来数据。
- 都以“规则策略 + 内部滚动评分 + 投票”作为核心架构。
- 都有一定可解释性，便于复盘。

两套方案差异点：

- 红球方案聚焦“命中任一胆码即算命中”，目标函数是二元命中。
- 蓝球方案聚焦“四码覆盖单蓝球”，本质是覆盖率问题。
- 蓝球方案额外引入了低中高三区覆盖约束，红球方案更偏结构投票。

## 四、改进原则

本次优化遵循以下原则：

1. **只使用公开历史开奖数据**，不引入主观选号。
2. **统一全量样本口径**，从 `dSsq/all_history_data.json` 读取 201 期历史。
3. **保留原方案可解释性**，不直接丢弃现有规则策略。
4. **引入轻量可学习模型**，把规则投票升级为“规则 + 号码级概率打分”的融合框架。
5. **全部采用严格前视回测**，并补充 Wilson 区间等统计验证。

## 五、优化后的统一模型架构

## 5.1 总体思路

新架构分为三层：

| 层级         | 作用                                         | 为什么这样做                                 |
| ------------ | -------------------------------------------- | -------------------------------------------- |
| 数据与特征层 | 统一校验数据、构造号码级特征                 | 先保证输入口径一致，否则任何模型比较都不可靠 |
| 规则策略层   | 保留原有六策略并继续做内部滚动评分           | 原方案的可解释性和经验积累仍有价值           |
| 融合评分层   | 用轻量逻辑回归做号码级打分，再与规则票数融合 | 既保留解释性，又让模型能学习特征组合关系     |

## 5.2 特征工程优化

### 红球号码级特征

针对每一期预测前，对每个候选红球 `01—33` 计算：

1. 最近 `5/10/20/50/100` 期出现频率。
2. 当前遗漏期数。
3. 与“适中遗漏区间”的距离得分。
4. 是否是上期重号。
5. 与上期号码集合的邻近程度。
6. 当前三区（01—11、12—22、23—33）在近12期内的回补偏差。
7. 奇偶平衡偏差。

这样做的原因是：旧方案中的“热号、遗漏、结构、回补、重号”原本分散在不同规则里，现在统一映射为可学习特征，便于模型从数据中自己学习权重。

### 蓝球号码级特征

针对每个候选蓝球 `01—16` 计算：

1. 最近 `3/6/12/24/48` 期频率。
2. 当前遗漏期数。
3. 适中遗漏得分。
4. 时间衰减得分。
5. 是否为上期开出号码。
6. 低中高分区偏差。
7. 奇偶平衡偏差。
8. 与最近 `6/12` 期蓝球均值的邻域接近度。
9. 候选号落在最近值域邻域内的强弱。
10. 与上期蓝球、近8期蓝球的连号距离和邻号密度。
11. 短窗与长窗频率差形成的冷热状态切换分。
12. 基于遗漏与短窗热度联合计算的“再激活”分。

这里保留了原蓝球方案中的热度、衰减、遗漏、分区思想，同时补进了更强的时序结构信息，不再把它们硬编码成最后的固定组合。

## 5.3 算法迭代

### 红球优化方案

- 保留原六大规则策略，继续计算最近12/24期滚动表现并做 3/2/1 投票。
- 新增号码级逻辑回归，对每个红球候选输出概率分数。
- 最终分数：

```text
FinalScore_red = 0.65 × ML概率分 + 0.35 × 规则投票分
```

- 取最终得分最高的 2 个号码作为红球胆码。

这样做的原因：

- 纯规则法容易受局部窗口扰动；
- 纯机器学习在小样本上容易发散；
- 融合法能兼顾稳定性与解释性。

### 蓝球优化方案

- 保留原六个蓝球规则策略，并继续做最近12/24期表现排序。
- 新增蓝球多项 softmax 概率模型，按整期 16 个候选号联合训练。
- 引入蓝球专用经验贝叶斯后验概率，作为更稳定的基础概率层。
- 将原“三区覆盖”从硬约束改为软性风险对冲项，只提供少量加分，不再强制占坑。
- 新增滚动参数搜索，自动评估贝叶斯窗口、先验强度与 ML 权重。
- 将最终输出从“线性分数排序”升级为 `softmax` 多分类概率分布，再取前4名作为围号。

最终分数：

```text
Score_blue(n) = BayesWeight × 贝叶斯后验分
              + MLWeight × ML概率分
              + LegacyWeight × 规则投票分
              + ZoneBonus

P_blue(n) = softmax(Score_blue(n) / Temperature)
```

当前默认稳态参数为：

```text
BayesWindow = 30
PriorStrength = 4.0
MLWeight = 0.15
LegacyWeight = 0.08
Temperature = 0.9
```

滚动参数搜索规则：

1. 只使用目标期之前的数据。
2. 在候选参数网格中，回看最近 `12/24/36` 个内部目标点。
3. 只有当候选参数在三个窗口上都显著优于默认稳态参数时，才允许切换。
4. 若没有足够强的证据，继续保留默认参数。

最终取多分类概率最高的 4 个号码，不再强制必须覆盖低中高三区。

这么改的原因有两点：

1. 实测发现硬性覆盖会显著拉低蓝球历史命中。
2. 蓝球更适合“平滑后的单号后验概率 + 轻量学习排序”结构，而不是让短窗投票长期主导结果。
3. 多分类概率输出比单纯排序更适合做后续校准、比较和参数搜索。

## 六、交叉融合方案

两套旧方案的优势分别是：

- 红球方案：结构解释强、重号/分区逻辑清楚。
- 蓝球方案：投票框架完整、滚动验证意识更强。

因此新融合方案采取“统一框架、分目标建模”的方式：

| 维度     | 融合做法                                              | 避免的问题             |
| -------- | ----------------------------------------------------- | ---------------------- |
| 数据口径 | 统一使用 `all_history_data.json` 全量历史             | 避免短样本结论不稳     |
| 排名逻辑 | 统一采用最近12/24期滚动评分                           | 保留旧方案的审计能力   |
| 学习能力 | 红球采用号码级逻辑回归，蓝球采用多项 softmax 概率模型 | 避免只靠手工规则       |
| 风险对冲 | 蓝球把分区覆盖降为软约束                              | 避免硬约束压制高分号码 |
| 统计验证 | 红蓝都输出窗口命中率 + Wilson 区间 + 随机基线         | 避免只看单一命中率     |

## 八、验证标准

优化后的方案不能只看“有没有改出新号码”，而要看是否满足以下验证标准：

### 8.1 数据合法性

- 期号严格升序。
- 红球每期 6 个、唯一、范围在 01—33。
- 蓝球范围在 01—16。

### 8.2 无前视泄漏

- 修改目标期之后的数据，目标期预测结果不得变化。

### 8.3 输出约束

- 红球输出必须为 2 个不同号码。
- 蓝球输出必须为 4 个不同号码。

### 8.4 回测结构一致性

- 20/38/60 期窗口的 `rows` 数量必须与窗口长度一致。
- `hits / total = hit_rate` 必须一致。

### 8.5 效果判断标准

建议以后把“是否继续沿用该模型”建立在以下规则上：

| 项目                | 建议阈值                                         |
| ------------------- | ------------------------------------------------ |
| 红球 38 期命中率    | 持续高于 33.5% 随机基线                          |
| 蓝球 38 期命中率    | 持续高于 25.0% 随机基线                          |
| 20 / 38 / 60 三窗口 | 至少 2 个窗口同时不低于基线                      |
| Wilson 区间         | 下限不要长期明显低于基线                         |
| 新旧模型对比        | 新模型至少不劣于旧模型，且不能靠改变验证口径实现 |

## 十、后续可继续推进的改进方向

这次已经把“规则方案”升级成了“规则 + 轻量学习 + 统一验证”的工程化版本，但还可以继续往下走：

### 10.1 样本量扩充

- 接入更完整的双色球历史开奖数据，不只保留近两年。
- 对接至少两个公开来源做交叉校验，降低单源抓取误差。

### 10.2 模型升级

- 红球可进一步尝试：
  - 梯度提升树
  - 排序学习模型
  - 基于组合层的多标签学习
- 蓝球可进一步尝试：
  - 多项分类概率模型
  - 隐马尔可夫或状态切换模型
  - 贝叶斯更新与分层先验

### 10.3 参数调优机制

- 通过时间切片交叉验证搜索：
  - 频率窗口长度
  - 融合权重
  - 适中遗漏中心值
  - 正则化强度

### 10.4 风险对冲补充
