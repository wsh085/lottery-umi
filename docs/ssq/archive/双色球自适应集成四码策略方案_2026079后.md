## 三、自适应集成四码模型

### 3.1 整体数据流

```text
目标期以前的历史数据
        ↓
六个固定基础策略分别预测4码
        ↓
使用目标期以前最近12期、24期做内部严格滚动评分
        ↓
按 0.7×近12期命中率 + 0.3×近24期命中率 排名
        ↓
前三名策略按3、2、1权重投票
        ↓
票数 → 六策略共识数 → 冷热混合分 → 号码升序裁决
        ↓
执行低中高三区覆盖，输出最终4码
```

### 3.2 公共定义

- 蓝球全集：01—16。
- 每个基础策略固定输出4个不同号码。
- 低位区：01—05；中位区：06—11；高位区：12—16。
- `F5、F12、F30`：号码在最近5、12、30期的出现频率。
- `Gap`：距最近一次开出的遗漏期数；最新一期出现记为0。
- 单期命中：当期实际蓝球包含在预测的4码中。
- 随机四码理论覆盖率：`4 ÷ 16 = 25%`。

### 3.3 六个基础策略

#### 策略A：近期热度

```text
S_hot(n) = 1.0×F5 + 0.7×F12 + 0.3×F30
```

作用：同时观察短、中、长三个频率窗口，近期信息权重最高。风险是短期追热。

#### 策略B：时间衰减

```text
S_decay(n) = Σ 0.88^age
```

`age=0`代表最新一期，越早的开奖贡献越小。作用是让模型自然响应近期变化，而不是突然切换固定窗口。

#### 策略C：适中遗漏

```text
S_gap(n) = exp(-|Gap-6|/4) + 0.25×F30
```

遗漏6期附近得分最高；遗漏过短或过长都会衰减，避免简单追热和盲目追极冷号。

#### 策略D：贝叶斯平滑

```text
Prior(n) = (全历史出现次数+1) ÷ (历史期数+16)
S_bayes(n) = (近30期出现次数 + 6×Prior) ÷ (近30期样本数+6)
```

作用：用全历史平滑近期30期频率，减少小样本中某个号码因偶然连开而得分过高。

#### 策略E：三区平衡

```text
S_zone(n) = 0.8×F12 + 0.4×F30 + 0.35×exp(-|Gap-6|/4)
```

先按上述分数排序，再确保4码覆盖低、中、高三个区域。它的目的只是控制组合集中度，不是假设分区存在轮换规律。

#### 策略F：冷热混合

```text
S_mix(n) = 0.6×S_hot(n) + 0.4×S_gap(n)
```

作用：在近期热度和适中遗漏之间折中，避免完全依赖单一特征。

### 3.4 内部策略排名

对任意目标期`t`：

1. 只使用`t`之前的数据。
2. 对六个基础策略分别复算`t`之前最近24个历史目标点。
3. 每个内部目标点同样只能读取该点之前的数据。
4. 统计最近12点和24点的命中率。
5. 固定按以下公式计算排名分：

```text
RankScore = 0.7×最近12期命中率 + 0.3×最近24期命中率
```

排名同分时，依次比较最近12期命中数、最近24期命中数以及预先固定的策略顺序，不能查看目标期真实蓝球。

### 3.5 3/2/1加权投票

- 第一名策略的每个候选号增加3票。
- 第二名策略的每个候选号增加2票。
- 第三名策略的每个候选号增加1票。
- 票数相同时，依次比较全部六策略的入选次数、冷热混合分和号码升序。
- 先保留排序最高的两个号码，再补足缺失的低、中、高区域，最后按总排序补足4码。

这一结构把“选择当前更有效的策略”和“避免四码全部集中在同一区域”分开处理，便于审计和复用。
